# Security Policy

## Supported Versions

Until the first formal tagged release, the supported security target is:
- `main`

## Reporting A Vulnerability

Please do **not** open a public issue with exploit details, secrets, or sensitive reproduction steps.

Preferred path:
- use GitHub's private vulnerability reporting / security advisory flow if available for this repository

Fallback path if private reporting is unavailable:
- open a minimal issue requesting a private security follow-up
- do **not** include exploit details, secrets, tokens, or sensitive payloads in that public issue

## What To Include

When reporting privately, include:
- affected area
- impact
- minimal reproduction
- whether secrets, auth, privacy, or IP boundaries are involved
- any known mitigation

## Response Goals

The project aims to:
- acknowledge valid reports promptly
- reproduce and triage the issue
- land a fix or mitigation when feasible
- communicate release/patch guidance when needed
